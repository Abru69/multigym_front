# Cómo Actualizar la Documentación

## Cuándo Actualizar

| Evento                    | Archivo a actualizar                          |
| ------------------------- | --------------------------------------------- |
| Completar un módulo       | `docs/PROJECT_STATUS.md`, `docs/CHANGELOG.md` |
| Encontrar bug conocido    | `docs/PROJECT_STATUS.md`                      |
| Agregar nueva dependencia | `docs/AI_CONTEXT.md`                          |
| Definir próxima tarea     | `docs/NEXT_SESSION.md`                        |
| Cambiar reglas            | `.ai/RULES.md`                                |

## Formato del Changelog

```markdown
## [YYYY-MM-DD]

### Completado

- Módulo X redesigned con Tailwind
- Componente Y creado

### En Progreso

- Módulo Z

### Bugs Conocidos

- Issue con W
```

## Reglas

1. Mantener los docs en **español**
2. Ser **conciso** (máximo 1 página por sección)
3. Incluir **comandos** relevantes
4. Referenciar **archivos** específicos
5. Actualizar después de **cada sesión** de trabajo
